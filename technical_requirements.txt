================================================================================
ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ
Проект: copilot-ollama-kie-proxy
================================================================================

ЦЕЛЬ
----
Создать асинхронное проксирующее приложение на Python, которое:
- Принимает запросы от VSCode (и любых Ollama-совместимых клиентов)
- Перенаправляет их на KIE.AI API (https://api.kie.ai/v1)
- Полностью эмулирует Ollama API, чтобы VSCode воспринимал сервис как локальный Ollama
- Дополнительно поддерживает OpenAI-совместимый API (/v1/) для работы с OpenAI SDK

Документация KIE.AI:
  https://kie.ai/claude-opus-4-6
  https://docs.kie.ai/market/claude/claude-opus-4-6


================================================================================
АРХИТЕКТУРА
================================================================================

Стек:
  - Python 3.9+
  - Виртуальное окружение venv (обязательно, изоляция зависимостей)
  - FastAPI + uvicorn (асинхронный веб-сервер)
  - httpx (асинхронный HTTP-клиент для проксирования)
  - python-dotenv (конфигурация через .env)

Виртуальное окружение:
  - Каталог venv/ в корне проекта (добавить в .gitignore)
  - Создаётся командой: python3 -m venv venv
  - Активация: source venv/bin/activate
  - Все make-команды (install, start, stop, ...) работают через venv/bin/python
    и venv/bin/pip -- без необходимости ручной активации.

Структура файлов:
  main.py            -- FastAPI-приложение, все эндпоинты и логика проксирования
  config.py          -- Settings-класс, читает переменные из .env
  logger.py          -- Настройка логирования (файлы, ротация)
  requirements.txt   -- Зависимости Python
  .env               -- Конфигурация (не коммитить в git)
  .env.example       -- Пример конфигурации (коммитить в git)
  Makefile           -- Команды управления сервисом
  start.sh           -- Скрипт запуска через uvicorn

================================================================================
API ЭНДПОИНТЫ -- OLLAMA-СОВМЕСТИМЫЕ (/api/)
================================================================================

GET  /api/version
  Возвращает версию Ollama (из OLLAMA_COMPAT_VERSION).
  Ответ: {"version": "0.8.0"}

GET  /api/tags
  Список доступных моделей. VSCode использует этот эндпоинт для обнаружения моделей.
  Ответ: {"models": [{"name": "claude-opus-4-6", "model": "claude-opus-4-6", ...}]}
  Каждая модель содержит поля: name, model, modified_at, size, digest, details.

POST /api/show
  Детали конкретной модели.
  Принимает name из JSON-тела, form-data или query-параметра.
  Ответ: {"modelfile": "...", "parameters": "...", "template": "...", "details": {...}}

POST /api/chat
  Чат с историей сообщений.
  Тело: {model, messages: [{role, content}], stream, temperature, top_p, top_k}
  stream=true  -- SSE-поток (каждая строка -- JSON-объект в формате Ollama)
  stream=false -- единый JSON-ответ

POST /api/generate
  Генерация текста по промпту (без истории).
  Тело: {model, prompt, stream, temperature, top_p, top_k}
  Поддерживает streaming аналогично /api/chat.

POST /api/chat/completions
  Алиас /api/chat для совместимости с некоторыми клиентами.

POST /api/pull
  Заглушка -- имитирует "скачивание" модели (200 OK, сообщение об успехе).

DELETE /api/delete
  Заглушка -- имитирует удаление модели.

HEAD /api/blobs/{digest}
  Заглушка -- отвечает 200 OK.

POST /api/embeddings
  Эмбеддинги текста. Проксирует на KIE.AI /embeddings.
  Ответ в формате Ollama: {"embedding": [...]}

GET  /api/health
  Healthcheck. Ответ: {"status": "ok"}

GET  /health
  Алиас /api/health.

GET  /
  Корневой маршрут. Возвращает минимальную информацию о сервисе.


================================================================================
API ЭНДПОИНТЫ -- OPENAI-СОВМЕСТИМЫЕ (/v1/)
================================================================================

Необходимы для работы с OpenAI Python SDK и VSCode-расширениями,
которые используют OpenAI-протокол.

GET  /v1/models
  Список моделей в формате OpenAI.
  Ответ: {"object": "list", "data": [{"id": "claude-opus-4-6", "object": "model", ...}]}

GET  /v1/models/{model}
  Детали конкретной модели в формате OpenAI.

POST /v1/chat/completions
  Чат в формате OpenAI. Трансформирует запрос для KIE.AI, ответ -- обратно.
  Поддерживает streaming (SSE).

POST /v1/completions
  Генерация текста в формате OpenAI. Оборачивает промпт в messages для KIE.AI.

POST /v1/embeddings
  Эмбеддинги в формате OpenAI. Проксирует напрямую.


================================================================================
ТРАНСФОРМАЦИЯ ЗАПРОСОВ
================================================================================

Входящий Ollama/OpenAI запрос -> KIE.AI формат:
  {
    "model": "<model>",
    "messages": [{"role": "...", "content": "..."}],
    "temperature": <float>,
    "top_p": <float>,
    "stream": <bool>
  }

KIE.AI ответ (не-streaming) -> Ollama-формат:
  {
    "model": "<model>",
    "created_at": "<ISO8601>",
    "message": {"role": "assistant", "content": "<text>"},
    "done": true,
    "total_duration": <ns>,
    "eval_count": <tokens>
  }

KIE.AI streaming ответ (SSE: data: {...}) -> каждый чанк в Ollama-формате:
  {"model": "...", "created_at": "...", "message": {"role": "assistant", "content": "<chunk>"}, "done": false}
  Финальная строка: {..., "done": true}


================================================================================
ЛОГИРОВАНИЕ
================================================================================

Три категории лог-файлов в LOG_DIR:
  general_YYYY-MM.log  -- общие события (старт/стоп, инициализация)
  error_YYYY-MM.log    -- ошибки и исключения
  request_YYYY-MM.log  -- все входящие запросы (если REQUEST_LOGGING_ENABLED=true)

Ротация: ежемесячная (новый файл каждый месяц).

Middleware логирует все входящие запросы:
  - method, path, query_params, client IP
  - заголовки (если REQUEST_LOG_HEADERS=true)
  - тело запроса до REQUEST_LOG_BODY_LIMIT символов (если REQUEST_LOG_BODY=true)

Каждое проксирование логирует:
  - тип операции (CHAT, GENERATE, EMBEDDINGS, ...)
  - модель, количество сообщений, streaming или нет

Ошибки логируют:
  - тип ошибки, сообщение, контекст запроса


================================================================================
ОБРАБОТКА ОШИБОК
================================================================================

- HTTP 4xx от KIE.AI -- прокинуть клиенту с тем же кодом
- HTTP 5xx от KIE.AI -- вернуть 502 Bad Gateway с деталями
- Сетевые ошибки (таймаут, connection refused) -- 503 Service Unavailable
- Невалидный JSON в ответе KIE.AI -- 502 с описанием
- Все необработанные исключения -- 500 + запись в error.log
- Формат ошибки: {"error": "<описание>"}


================================================================================
ПРОИЗВОДИТЕЛЬНОСТЬ
================================================================================

- Единый httpx.AsyncClient с connection pool (создаётся при startup, закрывается при shutdown)
- Таймаут на запросы к KIE.AI: 120 секунд (LLM-ответы длинные)
- Streaming через async generator: не буферировать весь ответ в памяти


================================================================================
СОВМЕСТИМОСТЬ С VSCODE
================================================================================

Требования для корректной работы VSCode с Ollama-расширениями:

1. Порт 11777 -- базовый порт прокси. В клиенте VSCode/Continue указать явно
   (http://127.0.0.1:11777), так как этот порт не сканируется автоматически.
2. GET /api/tags возвращает список моделей с полем "details" (family, format, ...)
3. GET /api/version возвращает версию >= 0.8.0
4. POST /api/chat поддерживает streaming в Ollama SSE-формате
5. POST /api/show возвращает modelfile с шаблоном промпта

Пример настройки VSCode (settings.json для Continue):
  "models": [{"title": "claude-opus-4-6", "provider": "ollama", "model": "claude-opus-4-6"}]

Минимальный набор curl-проверок:
  curl http://127.0.0.1:11434/api/version
  curl http://127.0.0.1:11434/api/tags
  curl http://127.0.0.1:11434/health

       -H "Content-Type: application/json" \
       -d '{"model":"claude-opus-4-6","messages":[{"role":"user","content":"ping"}],"stream":false}'

Python-тест с OpenAI SDK:
  from openai import OpenAI
  client = OpenAI(base_url="http://127.0.0.1:11434/v1/", api_key="ollama")
  response = client.chat.completions.create(
      model="claude-opus-4-6",
      messages=[{"role": "user", "content": "ping"}]
  )
