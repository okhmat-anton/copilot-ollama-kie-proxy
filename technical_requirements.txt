TECHNICAL REQUIREMENTS: Ollama-KIE.AI Proxy Service
================================================================================

PROJECT OVERVIEW
================================================================================
A lightweight Python proxy service that emulates the Ollama API to transparently
proxy requests to the KIE.AI API backend. This enables seamless integration of
KIE.AIs language model capabilities with existing Ollama-compatible clients,
including support for Claude Opus 4.6 models.

PURPOSE
-------
- Provide API compatibility layer between Ollama clients and KIE.AI backend
- Enable transparent model switching without client-side changes
- Support Claude Opus 4.6 and other advanced language models
- Maintain performance with minimal latency overhead
- Enable local development and testing environments

SYSTEM ARCHITECTURE
================================================================================
The proxy service will operate as a standalone Python application that:

1. Listens for incoming HTTP requests on a configurable port (default: 11434)
2. Accepts Ollama-compatible API requests (pull, generate, chat/completions)
3. Transforms requests into KIE.AI API format
4. Forwards requests to the KIE.AI backend API
5. Transforms responses back into Ollama API format
6. Returns responses to the client

CORE FUNCTIONALITY
================================================================================
The proxy must implement the following endpoints:

1. Model Management
   - GET /api/tags - List available models
   - POST /api/pull - Pull/download model (stub implementation)
   - DELETE /api/delete - Delete model (stub implementation)
   - HEAD /api/blobs/{digest} - Check blob existence

2. Model Inference
   - POST /api/generate - Generate text completion (streaming)
   - POST /api/chat/completions - Chat endpoint with messages

3. System Information
   - GET /api/version - Return service version
   - GET /health - Health check endpoint

API SPECIFICATIONS
================================================================================
Request Format (Ollama Compatible):
- Content-Type: application/json
- Support both streaming and non-streaming modes
- Accept streaming requests with "stream": true parameter
- Return Server-Sent Events (SSE) format for streaming responses

Response Format:
- JSON responses matching Ollama API specification
- Streaming responses in line-delimited JSON format
- Include proper HTTP status codes and error messages
- Support response metadata (model name, tokens, timing)

KIE.AI API Integration:
- Map Ollama model names to KIE.AI model identifiers
- Transform Ollama message format to KIE.AI request schema
- Extract relevant fields from KIE.AI responses
- Handle KIE.AI authentication and API keys

================================================================================

- Flask or FastAPI for HTTP server
- Requests library for backend communication

Configuration:
- Environment variables for KIE.AI API key
- Configurable proxy listen port
- Configurable backend URL
- Model mapping configuration (Ollama model name → KIE.AI model ID)

Request Handling:
- Request validation and schema verification
